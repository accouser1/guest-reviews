# Архитектура системы проверки гостей

## Обзор

Система представляет собой веб-приложение на базе FastAPI для автоматической проверки гостей отелей с расчетом риск-скоринга и управлением депозитами.

## Технологический стек

### Backend
- **FastAPI** - современный асинхронный веб-фреймворк
- **Python 3.11+** - язык программирования
- **SQLAlchemy 2.0** - ORM с поддержкой async/await
- **Alembic** - миграции базы данных
- **Pydantic** - валидация данных и схемы

### База данных
- **PostgreSQL 16** - основная реляционная БД
- **Redis 7** - кеширование и сессии

### Инфраструктура
- **Docker & Docker Compose** - контейнеризация
- **Uvicorn** - ASGI сервер
- **Nginx** - reverse proxy (production)

## Модели данных

### Основные сущности

#### User (Пользователь)
- Сотрудники отеля
- Роли: owner, manager, auditor
- Привязка к отелю

#### Hotel (Отель)
- Основная информация
- Настройки риск-скоринга
- Интеграции с платформами

#### Guest (Гость)
- Профиль гостя
- Контактные данные
- Уровень риска
- Флаг черного списка

#### Booking (Бронирование)
- Связь с гостем и отелем
- Даты заезда/выезда
- Статус брони
- Связь с риск-скорингом

#### Review (Отзыв)
- Оценка гостя (1-5)
- Комментарий
- Теги проблем
- Суммы ущерба

#### RiskScore (Риск-скоринг)
- Балл риска (0-100)
- Уровень риска
- Причины
- Рекомендация

#### Deposit (Депозит)
- Сумма депозита
- Статус
- Транзакции

### Связи между моделями

```
Hotel ──┬── User (1:N)
        ├── Booking (1:N)
        ├── Review (1:N)
        └── Settings (1:1)

Guest ──┬── Booking (1:N)
        ├── Review (1:N)
        └── RiskScore (1:N)

Booking ──┬── Guest (N:1)
          ├── Hotel (N:1)
          ├── RiskScore (1:1)
          ├── Deposit (1:1)
          └── Review (1:1)

Deposit ──── Transaction (1:N)
```

## Поток обработки брони

```
1. POST /bookings/
   │
   ├─> Найти или создать гостя
   │   (по телефону или email)
   │
   ├─> Создать запись брони
   │   (статус: pending_check)
   │
   ├─> Рассчитать риск-скоринг
   │   │
   │   ├─> Проверить черный список
   │   ├─> Собрать историю отзывов
   │   ├─> Рассчитать средний рейтинг
   │   ├─> Проверить историю повреждений
   │   └─> Вычислить итоговый балл
   │
   ├─> Принять решение
   │   │
   │   ├─> Балл < 40: APPROVE
   │   ├─> Балл 40-60: APPROVE (рассмотреть депозит)
   │   ├─> Балл 60-80: APPROVE_WITH_DEPOSIT
   │   └─> Балл > 80: REJECT
   │
   ├─> Создать депозит (если нужен)
   │   (30% от суммы брони)
   │
   └─> Вернуть результат
       (статус, риск, рекомендация, депозит)
```

## Алгоритм риск-скоринга

### Входные данные
- Профиль гостя
- История броней
- История отзывов
- Настройки отеля

### Расчет балла

```python
score = 50  # Базовый балл

# Черный список
if guest.blacklist_flag:
    score += 50

# Средняя оценка
if avg_rating < 3.0:
    score += 20

# История повреждений
if total_damage > 0:
    score += 15

# Новый гость
if booking_count == 0:
    score += 10

# Определение уровня
if score >= 80:
    level = CRITICAL
elif score >= 60:
    level = HIGH
elif score >= 40:
    level = MEDIUM
else:
    level = LOW
```

### Рекомендации

| Уровень | Действие |
|---------|----------|
| LOW | Заселить без депозита |
| MEDIUM | Рассмотреть депозит |
| HIGH | Требуется депозит 30% |
| CRITICAL | Отклонить бронь |

## Безопасность

### Аутентификация
- JWT токены
- Bcrypt для паролей
- Время жизни токена: 30 минут

### Авторизация
- Role-based access control (RBAC)
- Проверка принадлежности к отелю
- Разделение прав по ролям

### Защита данных
- HTTPS в production
- CORS настройки
- Rate limiting
- SQL injection защита (ORM)
- XSS защита (валидация)
