# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Что делает эта система?

Система автоматически проверяет гостей отеля при создании брони, рассчитывает риск-скоринг на основе истории отзывов и инцидентов, и принимает решение: заселить без депозита, заселить с депозитом или отклонить бронь.

### Как работает автоматическая проверка?

При создании брони система:
1. Находит или создает профиль гостя по телефону/email
2. Собирает всю историю отзывов о госте
3. Рассчитывает риск-скоринг (0-100 баллов)
4. Принимает решение на основе уровня риска
5. Создает депозит, если требуется

### Какие данные нужны для проверки гостя?

Минимально:
- Имя и фамилия
- Телефон или email (для поиска в базе)
- Даты заезда/выезда
- Сумма брони

## Риск-скоринг

### Как рассчитывается риск-скоринг?

Базовый балл 50, к которому добавляются:
- +50 если гость в черном списке
- +20 если средняя оценка < 3.0
- +15 если есть история повреждений
- +10 если новый гость без истории

### Что означают уровни риска?

- **LOW (0-40)**: Надежный гость, заселить без депозита
- **MEDIUM (40-60)**: Средний риск, рассмотреть депозит
- **HIGH (60-80)**: Высокий риск, требуется депозит 30%
- **CRITICAL (80+)**: Критический риск, отклонить бронь

### Можно ли настроить пороги риска?

Да, в настройках отеля (`HotelSettings`) можно изменить пороговые значения и процент депозита.

### Как гость попадает в черный список?

Вручную через API или панель управления. Обычно после серьезных инцидентов: кража, значительный ущерб, мошенничество.

## Депозиты

### Когда создается депозит?

Автоматически при уровне риска HIGH или CRITICAL (если не отклонена бронь).

### Какой размер депозита?

По умолчанию 30% от суммы брони. Можно настроить в `HotelSettings`.

### Как работает холдирование средств?

Система создает запись депозита со статусом `required`. Для реального холдирования нужна интеграция с платежной системой (Stripe, CloudPayments и т.д.).

### Что делать с депозитом после выезда?

- Если все в порядке: освободить депозит (`release`)
- Если есть ущерб: списать нужную сумму (`charge`)
- Можно частично списать (`partially_charged`)

## Отзывы

### Когда оставлять отзыв на гостя?

После выезда гостя (статус брони `checked_out`).

### Какие теги можно использовать?

- `theft` - Кража
- `property_damage` - Повреждение имущества
- `noise` - Шум
- `party` - Вечеринка
- `minibar_unpaid` - Неоплаченный минибар
- `fraud` - Мошенничество
- `rude_behavior` - Грубое поведение
- `payment_issue` - Проблемы с оплатой

### Влияют ли отзывы на будущие брони?

Да, отзывы участвуют в расчете риск-скоринга при следующих бронях этого гостя.

### Можно ли удалить отзыв?

Отзывы не удаляются, но можно изменить статус модерации на `hidden`.

## Интеграции

### Как интегрироваться с booking.com?

Нужно настроить webhook, который будет получать данные о новых бронях и автоматически создавать их в системе через API.

### Поддерживается ли Airbnb?

Архитектура позволяет интегрировать любую платформу через webhook или API.

### Как интегрировать платежную систему?

Нужно реализовать методы в `app/services/payment.py`:
- `hold_deposit()` - холдирование
- `charge_deposit()` - списание
- `release_deposit()` - возврат

## API

### Как получить токен авторизации?

```bash
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@grandhotel.ru","password":"admin123"}'
```

### Сколько действует токен?

30 минут. После истечения нужно получить новый токен.

### Как создать бронь через API?

```bash
curl -X POST http://localhost:8000/api/v1/bookings/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{...}'
```

См. полные примеры в `API_GUIDE.md` или `test_api.http`.

### Есть ли лимиты на запросы?

Да, 60 запросов в минуту на IP. Можно настроить в `RATE_LIMIT_PER_MINUTE`.

## Установка и настройка

### Какие требования к системе?

- Python 3.11+
- PostgreSQL 16
- Redis 7
- Docker (опционально)

### Как запустить локально?

См. подробную инструкцию в `SETUP.md`.

### Как изменить порт API?

В `.env` файле измените `API_PORT=8000` на нужный порт.

### Где хранятся логи?

В stdout. В production рекомендуется настроить сбор логов (ELK, Loki и т.д.).

## Безопасность

### Как защищены пароли?

Хешируются с помощью bcrypt перед сохранением в БД.

### Безопасны ли JWT токены?

Да, подписываются с помощью SECRET_KEY. Важно использовать длинный случайный ключ в production.

### Как защититься от SQL injection?

Используется SQLAlchemy ORM, который автоматически экранирует параметры.

### Нужен ли HTTPS?

В production обязательно. Настройте Nginx с SSL сертификатом.

## Производительность

### Сколько запросов в секунду выдерживает?

Зависит от железа. С базовой конфигурацией ~100-200 RPS. Можно масштабировать горизонтально.

### Как ускорить работу?

- Включить кеширование в Redis
- Добавить индексы в БД
- Использовать connection pooling
- Масштабировать горизонтально

### Можно ли использовать для большой сети отелей?

Да, архитектура поддерживает мультитенантность. Каждый отель изолирован.

## Проблемы и решения

### База данных не подключается

```bash
# Проверить статус
docker-compose ps postgres

# Проверить логи
docker-compose logs postgres

# Проверить переменные окружения
cat .env | grep DATABASE_URL
```

### API возвращает 401 Unauthorized

- Проверьте, что токен не истек (30 минут)
- Убедитесь, что токен передается в заголовке `Authorization: Bearer TOKEN`
- Проверьте, что SECRET_KEY не изменился

### Миграции не применяются

```bash
# Проверить текущую версию
alembic current

# Посмотреть историю
alembic history

# Применить принудительно
alembic upgrade head
```

### Docker контейнеры не запускаются

```bash
# Проверить логи
docker-compose logs

# Пересобрать
docker-compose down -v
docker-compose up --build
```
