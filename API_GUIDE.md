# Руководство по API системы проверки гостей

## Обзор

API предоставляет функционал для автоматической проверки гостей отеля, управления бронированиями, отзывами и риск-скорингом.

## Базовый URL

```
http://localhost:8000/api/v1
```

## Авторизация

Все защищенные endpoints требуют JWT токен в заголовке:

```
Authorization: Bearer YOUR_TOKEN
```

### Получение токена

**POST** `/auth/login`

```json
{
  "email": "admin@grandhotel.ru",
  "password": "admin123"
}
```

Ответ:
```json
{
  "access_token": "eyJhbGc...",
  "token_type": "bearer"
}
```

## Основной поток работы

### 1. Создание брони с автоматической проверкой

**POST** `/bookings/`

При создании брони система автоматически:
- Находит или создает профиль гостя
- Собирает историю отзывов
- Рассчитывает риск-скоринг
- Принимает решение о брони
- Создает депозит при необходимости

```json
{
  "hotel_id": "uuid",
  "source": "booking.com",
  "booking_external_id": "BK123456",
  "guest_first_name": "Иван",
  "guest_last_name": "Иванов",
  "guest_phone": "+79161234567",
  "guest_email": "ivan@example.com",
  "checkin_date": "2026-04-01",
  "checkout_date": "2026-04-05",
  "room_type": "Standard",
  "total_amount": 15000,
  "currency": "RUB"
}
```

Ответ:
```json
{
  "id": "booking-uuid",
  "status": "approved_with_deposit",
  "guest_id": "guest-uuid",
  "risk_level": "medium",
  "risk_score": 55,
  "recommendation": "APPROVE WITH DEPOSIT: Consider security deposit.",
  "deposit_required": true,
  "deposit_amount": 4500.00
}
```

### Статусы брони

- `new` - Новая бронь
- `pending_check` - Ожидает проверки
- `checked` - Проверена
- `approved` - Одобрена без депозита
- `approved_with_deposit` - Одобрена с депозитом
- `rejected` - Отклонена
- `checked_in` - Гость заселен
- `checked_out` - Гость выселен
- `cancelled` - Отменена

### 2. Просмотр деталей брони

**GET** `/bookings/{booking_id}`

Возвращает полную информацию о брони, включая риск-скоринг и депозит.

### 3. Просмотр профиля гостя

**GET** `/guests/{guest_id}`

```json
{
  "id": "guest-uuid",
  "first_name": "Иван",
  "last_name": "Иванов",
  "phone": "+79161234567",
  "email": "ivan@example.com",
  "risk_level": "medium",
  "blacklist_flag": false,
  "total_bookings": 5,
  "total_reviews": 3,
  "average_rating": 4.3
}
```

### 4. Создание отзыва после выезда

**POST** `/reviews/`

```json
{
  "booking_id": "booking-uuid",
  "rating": 4,
  "comment": "Хороший гость, все в порядке",
  "tags": [],
  "damage_amount": null,
  "unpaid_amount": null
}
```

### Теги отзывов

- `theft` - Кража
- `minibar_unpaid` - Неоплаченный минибар
- `noise` - Шум
- `party` - Вечеринка
- `property_damage` - Повреждение имущества
- `fraud` - Мошенничество
- `rude_behavior` - Грубое поведение
- `payment_issue` - Проблемы с оплатой

### 5. Просмотр истории отзывов гостя

**GET** `/reviews/guest/{guest_id}`

Возвращает все одобренные отзывы о госте.

## Риск-скоринг

### Логика расчета

Базовый балл: 50

Дополнительные баллы:
- В черном списке: +50
- Средняя оценка < 3.0: +20
- История повреждений: +15
- Новый гость (нет истории): +10

### Уровни риска

| Балл | Уровень | Решение |
|------|---------|---------|
| 0-40 | LOW | Заселить без депозита |
| 40-60 | MEDIUM | Рассмотреть депозит |
| 60-80 | HIGH | Требуется депозит (30%) |
| 80+ | CRITICAL | Отклонить бронь |

## Депозиты

При уровне риска MEDIUM или HIGH система автоматически создает депозит в размере 30% от суммы брони.

### Статусы депозита

- `not_required` - Не требуется
- `required` - Требуется
- `hold_pending` - Ожидает холдирования
- `held` - Захолдирован
- `partially_charged` - Частично списан
- `charged` - Списан
- `released` - Освобожден
- `failed` - Ошибка

## Роли пользователей

- `super_admin` - Суперадминистратор
- `owner` - Владелец отеля
- `manager` - Менеджер
- `auditor` - Аудитор (только чтение)

## Примеры использования

### Сценарий 1: Новая бронь от хорошего гостя

1. Создать бронь → Система находит гостя с хорошей историей
2. Риск-скоринг: LOW (балл 35)
3. Решение: `approved` без депозита
4. После выезда: оставить положительный отзыв

### Сценарий 2: Новая бронь от проблемного гостя

1. Создать бронь → Система находит гостя с плохими отзывами
2. Риск-скоринг: HIGH (балл 75)
3. Решение: `approved_with_deposit`, депозит 30%
4. После выезда: оставить отзыв с тегами проблем

### Сценарий 3: Новая бронь от гостя из черного списка

1. Создать бронь → Система находит гостя в blacklist
2. Риск-скоринг: CRITICAL (балл 100)
3. Решение: `rejected`
4. Бронь автоматически отклонена

## Коды ошибок

- `400` - Неверный запрос
- `401` - Не авторизован
- `403` - Доступ запрещен
- `404` - Не найдено
- `422` - Ошибка валидации
- `500` - Внутренняя ошибка сервера

## Лимиты

- Максимум 60 запросов в минуту на IP
- Размер файла: до 10 МБ
- Токен действителен 30 минут

## Webhook (планируется)

Для интеграции с booking.com, Airbnb и другими платформами можно настроить webhook для автоматического получения новых броней.
