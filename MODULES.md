# Модули системы проверки гостей

## Обзор модулей

Система состоит из 9 основных модулей, каждый из которых отвечает за определенную функциональность.

## 1. Авторизация (Authentication)

**Файлы:**
- `app/api/v1/endpoints/auth.py`
- `app/services/auth.py`
- `app/schemas/auth.py`

**Функции:**
- Регистрация пользователей
- Вход в систему (JWT токены)
- Хеширование паролей (bcrypt)
- Проверка токенов

**API Endpoints:**
- `POST /auth/login` - Вход
- `POST /auth/register` - Регистрация

**Роли пользователей:**
- `super_admin` - Суперадминистратор
- `owner` - Владелец отеля
- `manager` - Менеджер
- `auditor` - Аудитор (только чтение)

## 2. Управление отелями (Hotels)

**Файлы:**
- `app/models/hotel.py`

**Сущности:**
- `Hotel` - Основная информация об отеле
- `HotelBranch` - Филиалы отеля
- `HotelSettings` - Настройки риск-скоринга и депозитов
- `Integration` - Интеграции с booking.com, Airbnb и т.д.

**Функции:**
- Управление профилем отеля
- Настройка правил риск-скоринга
- Настройка политики депозитов
- Управление интеграциями

## 3. Бронирования (Bookings)

**Файлы:**
- `app/api/v1/endpoints/bookings.py`
- `app/services/booking.py`
- `app/models/booking.py`
- `app/schemas/booking.py`

**Функции:**
- Создание брони с автоматической проверкой
- Поиск или создание профиля гостя
- Запуск риск-скоринга
- Принятие решения (одобрить/депозит/отклонить)
- Создание депозита при необходимости
- Просмотр истории броней

**API Endpoints:**
- `POST /bookings/` - Создать бронь
- `GET /bookings/{id}` - Детали брони
- `GET /bookings/` - Список броней

**Статусы брони:**
- `new` - Новая
- `pending_check` - Ожидает проверки
- `checked` - Проверена
- `approved` - Одобрена
- `approved_with_deposit` - Одобрена с депозитом
- `rejected` - Отклонена
- `checked_in` - Заселен
- `checked_out` - Выселен
- `cancelled` - Отменена

## 4. Карточка гостя (Guest Profile)

**Файлы:**
- `app/api/v1/endpoints/guests.py`
- `app/services/guest.py`
- `app/models/guest.py`
- `app/schemas/guest.py`

**Функции:**
- Автоматическое создание профиля при первой брони
- Поиск гостя по телефону или email
- Просмотр полной истории гостя
- Статистика: количество броней, средний рейтинг
- Управление черным списком

**API Endpoints:**
- `GET /guests/{id}` - Профиль гостя
- `GET /guests/` - Список гостей

**Данные профиля:**
- ФИО, контакты
- Уровень риска
- Флаг черного списка
- Количество броней
- Средняя оценка
- История отзывов

## 5. Отзывы на гостей (Reviews)

**Файлы:**
- `app/api/v1/endpoints/reviews.py`
- `app/models/review.py`
- `app/schemas/review.py`

**Функции:**
- Создание отзыва после выезда
- Оценка гостя (1-5 звезд)
- Теги проблем (шум, повреждения и т.д.)
- Фиксация сумм ущерба
- Модерация отзывов

**API Endpoints:**
- `POST /reviews/` - Создать отзыв
- `GET /reviews/guest/{guest_id}` - Отзывы о госте

**Теги отзывов:**
- `theft` - Кража
- `minibar_unpaid` - Неоплаченный минибар
- `noise` - Шум
- `party` - Вечеринка
- `property_damage` - Повреждение имущества
- `fraud` - Мошенничество
- `rude_behavior` - Грубое поведение
- `payment_issue` - Проблемы с оплатой

## 6. Риск-скоринг (Risk Scoring)

**Файлы:**
- `app/services/risk_scoring.py`
- `app/models/risk_score.py`

**Функции:**
- Автоматический расчет при создании брони
- Анализ истории отзывов
- Проверка черного списка
- Расчет среднего рейтинга
- Анализ истории повреждений
- Генерация рекомендации

**Алгоритм:**
```
Базовый балл: 50

+ 50 если в черном списке
+ 20 если средняя оценка < 3.0
+ 15 если есть история повреждений
+ 10 если новый гость без истории

Уровни:
0-40: LOW (заселить)
40-60: MEDIUM (рассмотреть депозит)
60-80: HIGH (требуется депозит)
80+: CRITICAL (отклонить)
```

**Рекомендации:**
- `APPROVE` - Заселить без депозита
- `APPROVE WITH DEPOSIT` - Требуется депозит
- `REJECT` - Отклонить бронь

## 7. Депозиты (Deposits)

**Файлы:**
- `app/models/deposit.py`

**Функции:**
- Автоматическое создание при высоком риске
- Расчет суммы (30% от брони по умолчанию)
- Холдирование средств
- Списание при ущербе
- Возврат после выезда
- История транзакций

**Статусы:**
- `not_required` - Не требуется
- `required` - Требуется
- `hold_pending` - Ожидает холдирования
- `held` - Захолдирован
- `partially_charged` - Частично списан
- `charged` - Списан
- `released` - Освобожден
- `failed` - Ошибка

**Типы транзакций:**
- `hold` - Холдирование
- `charge` - Списание
- `release` - Возврат

## 8. Уведомления (Notifications)

**Файлы:**
- `app/models/notification.py`

**Функции:**
- Уведомления о новых бронях
- Алерты о высоком риске
- Напоминания о депозитах
- Уведомления о новых отзывах

**Каналы:**
- Email
- SMS (планируется)
- Push (планируется)
- In-app

## 9. Аналитика (Analytics)

**Планируется:**
- Статистика по бронированиям
- Распределение по уровням риска
- Эффективность депозитов
- Топ проблемных гостей
- Динамика отзывов
- Экспорт отчетов

## Основной поток

1. **Создание брони** → Модуль Bookings
2. **Поиск гостя** → Модуль Guest
3. **Сбор отзывов** → Модуль Reviews
4. **Расчет риска** → Модуль Risk Scoring
5. **Создание депозита** → Модуль Deposits (если нужен)
6. **Уведомление** → Модуль Notifications
7. **После выезда** → Модуль Reviews (создание отзыва)
